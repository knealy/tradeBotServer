✅ COMPLETED: Breakeven flag for stop_bracket command

Implementation:
1. Added `--breakeven` flag to stop_bracket command
   Usage: stop_bracket MNQ BUY 1 25800 25750 25900 --breakeven
   
2. Added env variables for control:
   - MANUAL_BREAKEVEN_ENABLED=true/false (default: false)
     * When true, breakeven is enabled by default for all manual orders
     * --breakeven flag overrides this setting
   
   - MANUAL_BREAKEVEN_PROFIT_POINTS=15.0 (default: 15.0)
     * Profit threshold in points before moving stop to breakeven
     * Same auto-start/stop logic as overnight strategy

3. How it works:
   - Order placed with stop entry + SL/TP brackets
   - If breakeven enabled, monitoring starts automatically
   - When position reaches +15pts profit (or configured threshold)
   - Stop loss automatically moves to entry price (breakeven)
   - Monitoring auto-stops when:
     * Breakeven is triggered
     * Position is closed (TP or SL hit)

4. Example usage:
   # With flag:
   > stop_bracket MNQ BUY 1 25800.00 25750.00 25900.00 --breakeven
   
   # With env variable (enables by default):
   MANUAL_BREAKEVEN_ENABLED=true
   > stop_bracket MNQ BUY 1 25800.00 25750.00 25900.00
   
5. Output shows breakeven status:
   ```
   ⚠️  CONFIRM STOP BRACKET ORDER:
      Symbol: MNQ
      Side: BUY
      Quantity: 1
      Entry (Stop) Price: $25800.0
      Stop Loss: $25750.0
      Take Profit: $25900.0
      Breakeven: ✅ Enabled
      Breakeven Threshold: 15.0 points profit
   ```
